# 🔧 07 - Troubleshooting

## 📌 Overview

This section documents troubleshooting scenarios encountered across the homelab, covering:

* Physical layer (cabling)
* Network layer (IP, VLANs, routing)
* Virtualisation layer (Proxmox, Ubuntu VMs)
* System and service-level issues (SSH, DNS)

The goal is to demonstrate a structured, real-world troubleshooting methodology used in data centre and network environments.


---

## 🧭 Troubleshooting Approach

A layered approach is used to isolate and resolve issues:

1. **Layer 1 – Physical**

   * Check cables, ports, link lights

2. **Layer 2 – Data Link**

   * Verify VLAN assignments, MAC connectivity

3. **Layer 3 – Network**

   * Validate IP addressing, routing, gateway

4. **Layer 4–7 – Services**

   * Check SSH, DNS, and running services

* Testing assumptions step-by-step
